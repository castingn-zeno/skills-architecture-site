How we work · 2026

요청 한 줄이
통합 산출물이 되는
7단계 루프.

캐스팅엔 마케팅팀은 사람을 늘리는 대신 Claude 스킬을 팀 단위로 설계해 병렬로 돌립니다. 모든 요청은 동일한 프로세스를 타고, 협업 라우팅이 강제되며, 결정은 메모리에 쌓여 다음 루프의 연료가 됩니다.

20 마운트 스킬 7 마케팅 팀 7 개발 팀 1 구매 고문 (김 이사) 22 고객 회의록 1 리더
The 7-step execution loop

요청 → 산출물.
7단계 업무 실행 프로세스.

모든 요청은 같은 흐름을 탑니다. 아래 예시는 "제노 랜딩페이지 새로 만들어줘"가 어떻게 통합 산출물이 되는지의 실제 추적.

👤
사용자 요청 도착 "제노 랜딩페이지 새로 만들어줘"
🧭
Step 00 · ops-router (0번 스킬)

난이도 판별 → 모델 선택 → 예산 결정

모든 요청의 최초 발동 스킬. 사고 깊이·맥락 크기·리스크를 3축 스코어링 해서 난이도를 정하고, 거기에 맞는 모델과 토큰 예산을 고릅니다. 결정은 1줄로 다음 스킬에 핸드오프.

Low · haiku조사·수집·단순 요약
Mid · sonnet작성·편집·리뷰·구조화
High · opus전략·리브랜딩·복합 협업
// ops-router 핸드오프 선언 (1줄)
[ops-router → content-team]
난이도: High / 모델: opus / 예산: loose / 협업: brand · customer · design · seo · aeo · dev-*
Grep > Glob > Read (파일 크기 모르면 Grep부터)
Read는 offset/limit 필수, 중복 읽기 금지
설명 ≤ 20% / 산출물 ≥ 80%
사용자가 명시한 모델·스킬은 존중
🎯
Step 01

트리거 매칭 + 진입 스킬 결정

description에 등록된 키워드와 사용자 요청을 비교해 가장 적합한 진입 스킬 1개를 선정합니다.

키워드: "랜딩페이지" 키워드: "제노" → 진입: content-team
📚
Step 02

컨텍스트 로드

메모리 · 고객 회의록 · 프로젝트 규칙 중 이번 과제에 필요한 것만 선택적으로 불러옵니다.

메모리MEMORY.md + 25개 파일에서 관련 프로젝트만 선택
회의록 22건/14_skill/customer/ — 고객 언어 채굴 원천
CLAUDE.md데일리 로그 규칙 · 한글 출력 · 금지어
🤝
Step 03

협업 라우팅 결정

진입 스킬이 작업 시작 전에 어느 팀과 협업할지 1줄로 명시합니다. 누락된 팀이 있으면 가짜 산출물로 간주.

// content-team 작업 시작 시 라우팅 선언
routing = brand-team(가드레일) + customer-team(고객 언어 · 회의록) +
          strategy-team(메시지 우선순위) + seo-team(키워드 · 메타) +
          aeo-team(FAQ 형식) + design-team(비주얼 사양) +
          dev-frontend(인터랙션) + dev-backend(폼 · CRM 연동)
⚙️
Step 04

작업 모드 선택 + 실행

각 스킬의 표준 모드 중 1개를 골라 실행. 여기선 content-team Mode 5 (Landing Page).

자료 소집회의록 · 메모리
구조 잡기섹션 · 그리드
본문 작성카피 · CTA
자가 검수가드레일 체크
✅
Step 05

가드레일 자가 체크리스트

배포 전 6개 항목을 스스로 검수. 하나라도 실패하면 Step 4로 되돌아갑니다.

ZENO/제노/캐스팅엔 표기 정확?
CFO 금지어("감으로 산다") 사용 여부?
보일러플레이트 순서 (제노 앞/업무마켓9 뒤)?
토스 UX 라이팅 8원칙 통과?
출처 주석 (회의록·통계)?
채널 글자 수 / SEO·AEO 형식?
📦
Step 06

산출물 출력 + 핸드오프 사양

본문 + 근거 주석 + 다음 팀이 받아 쓸 수 있는 사양을 함께 내보냅니다.

→ dev-frontend카피 + 인터랙션 사양표 (스크롤·폼·캐러셀)
→ design-teamHero 시각 사양 · 8pt 그리드 · 반응형
→ seo/aeo메타 · JSON-LD · FAQ Q&A
💾
Step 07

영속화 — 메모리 + 데일리 로그

새 인사이트·피드백은 메모리에 반영되고, 업무는 데일리 로그에 Why/What/How로 남습니다.

📓 데일리 로그 ~/.claude/daily-logs/YYYY-MM-DD.md

Why / What / How 3축 필수. "오늘 정리" 한 마디로 Notion DB 동기화.

🧠 메모리 ~/.claude/projects/…/memory/

project / feedback / user / reference 4종. MEMORY.md 인덱스 자동 주입.

✓
통합 산출물 카피 + 구조 + 인터랙션 사양 + 디자인 사양 + SEO/AEO 사양 + 핸드오프 명세

content-team이 진입 스킬이 되고, 8개 팀이 병렬로 협업하는 실제 흐름. 각 팀이 무엇을 공급했는지까지 추적.

Core principles

모든 스킬이 지키는 7가지 원칙

팀이 달라도, 과제가 달라도, 이 7개는 공통입니다. 이게 지켜지지 않으면 출시되지 않습니다.

1

메모리 우선

추측 금지. 메모리·1차 자료 먼저.

2

협업 라우팅

혼자 작업 금지. 1줄 명시 의무.

3

출처 주석

인용·통계는 반드시 출처 표기.

4

가드레일

brand-team 표기·금지어 양보 X.

5

핸드오프

다음 팀이 받아 쓸 수 있는 사양으로.

6

영속화

데일리 로그 + 메모리 갱신.

7

한글 출력

모든 산출물은 한글로.

Team dependency graph

팀 간 의존성 그래프

누가 누구에게 공급하는지, 어떤 순서로 신호가 흐르는지. 위에서 아래로 읽으면 한 번의 출시가 보입니다. 왼쪽 procurement-advisor(김 이사)는 구매 도메인 고문으로, 전략·콘텐츠·PR·서비스기획에 인사이트를 주입합니다(골드 점선).

Why this structure wins

이 구조의 강점

사람이 많아져야 가능했던 것을, 사람을 늘리지 않고 가능하게 만듭니다.

⚡️

속도

라우팅 → 병렬 작업 → 자가 검수가 한 루프 안에서 끝납니다. 회의 없이도 통합 산출물이 나옵니다.

🎯

일관성

brand-team 가드레일이 모든 출구를 통과. 같은 과제를 열 번 해도 같은 톤·표기로 나옵니다.

🧠

누적

한 번 한 일은 메모리로 쌓이고, 다음 루프의 연료가 됩니다. 시간이 지날수록 빨라집니다.

Tooling

루프를 유지시켜주는 세 가지 도구

사람이 기억할 필요가 없도록, 시스템이 기억합니다.

SKILL.md

~/.claude/skills/<team>/SKILL.md

각 팀의 역할·모드·협업 라우팅을 YAML + 마크다운으로 정의. 저장과 동시에 hot-reload.

Memory System

~/.claude/projects/…/memory/

project / feedback / user / reference 4종 메모리. MEMORY.md 인덱스가 매 대화에 자동 주입.

~/.claude/daily-logs/YYYY-MM-DD.md
